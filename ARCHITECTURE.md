# ChatSphere Architecture

## Executive Summary

ChatSphere is a full-stack AI-native chat platform built with React 18, TypeScript, Vite, Express, MongoDB, Socket.IO, and a multi-provider AI gateway. It supports solo AI conversations, real-time group rooms, and enterprise features like admin dashboards, analytics, and memory systems.

### Token Strategy
- **Access Token**: Short-lived (15 min), sent in `Authorization: Bearer` header
- **Refresh Token**: Long-lived (7 days), stored in httpOnly cookie
- **Rotation**: Refresh token rotated on each use, old token blacklisted

## Scalability Considerations

### Current Limitations
1. **Single-server Socket.IO**: No Redis adapter for multi-instance
2. **In-memory state**: Room users, typing state, flood control in memory
3. **No caching**: No Redis/Memcached for frequent queries
4. **No message queue**: AI requests are synchronous
5. **No CDN**: Static assets served from same origin

### Scaling Path
1. **Horizontal**: Add Redis adapter for Socket.IO, sticky sessions
2. **Caching**: Add Redis for session, model catalogs, frequent queries
3. **Queue**: Add Bull/BullMQ for AI request processing
4. **CDN**: Serve frontend via CloudFront/Cloudflare
5. **Database**: MongoDB sharding for message collection
6. **Microservices**: Extract AI gateway, auth, chat into services

---

## Monitoring & Observability

### Current State
- Console logging with structured format
